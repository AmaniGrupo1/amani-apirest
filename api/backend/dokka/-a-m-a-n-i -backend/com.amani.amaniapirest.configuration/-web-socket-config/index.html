Configuracion de WebSocket con STOMP sobre SockJS.

Expone el endpoint /ws para las conexiones WebSocket y configura un broker de mensajes en memoria con los destinos /topic (difusion) y /queue (punto a punto). Los mensajes de la aplicacion se envian al prefijo /app.
